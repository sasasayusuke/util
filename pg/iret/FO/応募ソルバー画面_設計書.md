# 応募ソルバー画面 設計書

## 関連課題

* なし

## 機能概要

人材要件に対する応募ソルバーを管理する画面。
既存の個人ソルバーやソルバー候補から選択、または新規でソルバー候補を追加することができる。

## 画面デザイン

プレゼンテーション1.pdf

## 利用可能権限

| ロール | 条件 |
|--------|------|
| APT | 全機能利用可能 |
| 法人ソルバー | 自社に関連する応募ソルバーの管理が可能 |
| 個人ソルバー | アクセス不可 |

## シーケンス・フロー

### メインフロー
1. 人材要件詳細画面から「変更する」ボタンクリック
2. 応募ソルバー画面へ遷移（人材要件IDを受け取る）
3. 初期表示処理
   - APIコール
   - 既存の応募ソルバー情報を取得
   - 画面を表示
4. ソルバーの管理
   - ＋ボタンでポップアップからソルバーを追加
   - －ボタンで選択したソルバーを削除
   - 編集ボタンで別タブで詳細
5. 「変更する」ボタンクリック
   - 更新内容をAPIに送信
   - 成功時は完了メッセージ表示

### 個人ソルバー追加フロー
1. 「＋」ボタンクリック（個人ソルバー管理エリア）
2. 個人ソルバー追加ポップアップ表示
3. プルダウンから登録済み個人ソルバーを選択
4. 「追加」ボタンクリック
5. メイン画面の個人ソルバー一覧に追加

### ソルバー候補追加フロー
1. 「＋」ボタンクリック（ソルバー候補管理エリア）
2. ソルバー候補追加ポップアップ表示
3. プルダウンから登録済みソルバー候補を選択
4. 「追加」ボタンクリック
5. メイン画面のソルバー候補一覧に追加
