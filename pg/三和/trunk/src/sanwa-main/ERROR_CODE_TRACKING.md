## 次回使用予定番号

**013** から開始してください

## エラーコード追加手順

1. **番号確認**: 上記台帳で最新番号を確認
2. **カテゴリ選択**: 適切なカテゴリ（AUTH/DB/API/UI/SYS/GEN）を選択
3. **番号割当**: 次の連番を割当
4. **実装**: ファイルにエラーコードを追加
5. **台帳更新**: この台帳に記録
6. **定義書更新**: ERROR_CODES.md に追加

## カテゴリ別使用状況

- **AUTH (認証・認可)**: 001-003 (3個使用)
- **DB (データベース)**: 004-006 (3個使用)  
- **API (通信)**: 007-008 (2個使用)
- **UI (フロントエンド)**: 009-011 (3個使用)
- **SYS (システム)**: 未使用
- **GEN (一般)**: 012 (1個使用)

## 注意事項

- **グローバル一意性**: すべてのエラーコードは全システムでユニーク
- **連番管理**: 欠番は作らず、必ず連番で追加
- **台帳更新**: 新しいエラーコードを追加したら必ずこの台帳を更新
- **削除禁止**: 一度使用したエラーコードは削除しない（ログとの整合性保持）
